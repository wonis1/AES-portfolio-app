# AI 기반 실시간 교통·식당 추천 챗봇 구축 (Internal PoC)

## 📌 프로젝트 개요

자사 AI 챗봇 플랫폼(CogInsight)을 활용하여  
실시간 공공 데이터 연동 기반 대화형 서비스를 설계·구현한 프로젝트.

요구사항 정의부터 연동 설계, 상태 관리 구조 설계, 테스트 및 문서화까지  
전 과정을 단독 수행.

- 형태: 사내 PoC (단독 수행)
- 역할: 기획 · 설계 · 구현 · 테스트 · 문서화
- 목적: 실서비스 수준의 대화형 아키텍처 설계 및 플랫폼 이해 고도화

---

## 🚀 주요 구현 내용

### 1️⃣ 실시간 교통 정보 연동 구조 설계

- 버스 / 지하철 도메인 분리 기반 Intent · Entity 모델링
- 공공 Open API 연동 및 응답 데이터 구조 표준화
- 성공 / 실패 / 예외 코드에 따른 다단계 분기 설계
- 외부 연동 실패 시 사용자 이탈 최소화를 위한 대체 흐름 설계

> 단순 데이터 출력이 아닌, API 상태를 포함한 UX 재설계에 초점

---

### 2️⃣ 도메인 기반 추천 로직 설계

- 메뉴 / 식당 엔티티 분리 모델링
- 엔티티 + 랜덤 조건 결합 기반 추천 분기 구조 설계
- 식당 리스트 카테고리 정렬 구조 구현
- 버튼 중심 흐름에서 자연어 기반 흐름으로 전환

---

### 3️⃣ 상태 기반 대화 아키텍처 설계

- 세션 컨텍스트 기반 사용자 상태 추적 구조 설계
- 반복 입력 및 Intent 미매칭 구간의 단계적 전환 로직 구성
- 강제 Postback 흐름 제거 및 선택 기반 다이얼로그 재구성
- 예외 처리 로직을 메인 플로우와 분리하여 구조 단순화

> 분기문 중심이 아닌 **상태 기반 서비스 흐름**으로 재설계

---

### 4️⃣ FAQ 응답 구조 개선

- ESD 기반 일괄 응답 처리 구조 설계
- Output 중심 응답 구조 정렬
- 불필요한 사용자 입력 기반 ESD 제거

---

## 📝 설계 산출물

문서 기반 개발 프로세스 적용

- 요구사항 정의서
- 기능 명세서
- REST API 연동 정의서
- 성공 / 실패 분기 설계 문서
- WBS 및 진행 상태 관리표
- API 테스트 명세 (Postman 기반)
- 운영 가이드 초안

---

## 🔎 테스트 및 검증

- 실시간 공공 API 응답 기반 연동 검증
- 실패 / 타임아웃 / 예외 코드별 시나리오 테스트
- 사용자 테스트 배포 및 피드백 반영
- 예외 케이스 재정의 및 UX 개선 반복 적용

## 🛠 기술 스택

- CogInsight
- REST API

- JSON Parsing
- Postman
- 상태 기반 시나리오 설계
- WBS, 요구사항정의서, RFP, 연동정의서 관리

---

## 💡 핵심 경험

- 요구사항 → 설계 → 연동 → 테스트 → 문서화 **풀사이클 수행**
- API 연동 실패까지 고려한 UX 설계 경험
- Intent / Entity 모델링이 사용자 경험에 미치는 영향 체감
- 문서 기반 개발 프로세스 적용 경험
