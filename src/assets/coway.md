# 📌 코웨이 챗봇 운영 및 고도화 프로젝트

**기간**: 2024.05 ~ 2025.07 (약 14개월)  
**역할**: 전담 운영 담당 (구조 이해 및 기능 확장 수행)

---

## 🧩 프로젝트 개요

실 운영 중인 B2C 챗봇을 전담 운영하며  
다이얼로그 구조, NLP 구성, 외부 API 연동, 통계 구조까지 전반을 관리했습니다.

단순 요청 처리 수준이 아니라,  
요구사항 분석 → 시나리오 설계 → 구현 → 검증 → 배포까지 직접 수행했습니다.

---

## 🏗 시스템 구조 이해 및 운영

### 1️⃣ 다이얼로그 구조 관리

- 대분류 기반 메인 시나리오 구조 운영
- 조건 분기 로직 설계 및 수정
- QnA형 단답 구조 및 예외 처리 로직 관리
- 신규 요구사항 반영 및 기존 플로우 개선

운영 환경 특성상 작은 분기 오류도 사용자 이탈로 이어지기 때문에,  
시나리오 구조를 이해한 상태에서 안정적으로 확장했습니다.

---

### 2️⃣ 문제 해결 로직 운영

제품별 문제 해결 영역은 단순 텍스트 응답이 아닌,

- 제품 데이터 매핑
- 문제 유형별 조건 분기
- 예외 및 fallback 처리

구조로 설계되어 있었습니다.

신규 제품 및 문제 유형 추가 시:

- 데이터 매핑 보완
- 분기 로직 확장
- 누락 케이스 수정

을 통해 커버리지를 지속적으로 보완했습니다.

---

### 3️⃣ 제품 추천 기능 (외부 API 연동)

제품 추천 기능은 다음 흐름으로 구성되어 있습니다:

카테고리 매칭 → 제품 코드 추출 → 외부 API 호출 → 응답 데이터 가공 → 캐로셀 UI 구성

- 제품 코드 기반 REST API 연동
- 응답 데이터 가공 및 구조화
- HTML / CSS / JavaScript 기반 동적 UI 구현
- 이벤트 영역 캐로셀 커스터마이징 대응

단순 UI 수정이 아니라, 데이터 흐름을 이해한 상태에서 화면을 구성했습니다.

---

## 🧠 NLP 운영 및 품질 유지

- Intent / Entity 정의 및 보완
- 신규 발화 유형 반영
- 오분류 케이스 수정
- 운영 로그 기반 개선

실 운영 챗봇은 분류 정확도가 곧 서비스 품질로 이어집니다.  
메시지 로그를 기반으로 분류 품질을 유지하며, 지속적으로 발화 패턴을 반영했습니다.

---

## 📊 사용자 행동 분석 및 통계 구조 개선

운영 중 고객사는 챗봇 이탈 지점 분석에 높은 관심을 보였습니다.

이에 따라:

- 주요 응답 지점 통계 구조 정비
- 이탈 위치 추적 가능하도록 개선
- 메시지 로그 기반 이슈 분석 대응

단순 기능 수정이 아닌,  
챗봇 사용 흐름을 분석하고 개선하는 작업을 수행했습니다.

---

## 🔧 UX 안정성 개선 사례

비인증 환경에서 앱 전환 또는 뒤로가기 시  
세션이 종료되는 이슈가 발생했습니다.

원인 분석 후:

- 재접속 시 대화 이력 유지 구조 적용
- 사용자 경험(UX) 관점에서 안정성 개선

운영 환경에서 실제 사용 시나리오를 고려한 개선 경험입니다.

운영 서비스 특성상 안정적인 배포 프로세스를 유지했습니다.
